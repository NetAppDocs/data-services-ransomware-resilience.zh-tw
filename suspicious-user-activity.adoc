= 在 NetApp Ransomware Resilience 中設定使用者活動偵測
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
NetApp Ransomware Resilience 使用者行為偵測可協助您防止使用者層級的勒索軟體事件。若要在 Ransomware Resilience 中啟用使用者行為偵測，您必須安裝至少一個使用者活動代理程式，該代理程式會建立資料收集環境，以監控使用者行為中類似勒索軟體事件的異常模式。

使用者活動代理程式包含資料收集器和使用者目錄連接器，它們都會將資料傳送到 SaaS 位置進行分析。

* **資料收集器**會從 ONTAP 收集使用者活動資料。當您建立包含使用者行為偵測的保護策略時，系統會自動建立資料收集器。
* **使用者目錄連接器**用於連接到您的目錄，並將使用者 ID 對應到使用者名稱。您必須配置使用者目錄連接器。


使用者活動代理程式、資料收集器和使用者目錄連接器都可以從 Ransomware Resilience 設定儀表板進行管理。


NOTE: 如果您已在使用 NetApp Data Infrastructure Insights（DII）Workload Security、建議您將相同的 Workload Security 代理程式用於 Ransomware Resilience。您無需為 Ransomware Resilience 單獨部署 Workload Security 代理程式、但使用相同的 Workload Security 代理程式需要 Ransomware Resilience Console 組織與 DII Storage Workload Security 租戶建立配對關係。請聯絡您的客戶代表以啟用此配對。

如果您未使用 DII，請按照此處的組態說明進行操作。



[[before-you-begin]]
== 開始之前

* 確保您符合link:reference-user-activity-requirements.html["作業系統、伺服器和網路要求"]。


*所需 Console 角色* 若要啟用使用者行為偵測，您需要 **Organization admin role**。對於後續的使用者行為偵測配置，您需要 **Ransomware Resilience user behavior admin role**。link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-ransomware-roles.html["了解NetApp Console的勒索軟體復原角色"^]。

確保每個角色都在組織層級套用。



[[create-a-user-activity-agent]]
== 建立使用者活動代理程式

使用者活動代理程式是 link:#add-a-data-collector["資料收集器"] 的可執行環境；資料收集器與 Ransomware Resilience 共用使用者活動事件。您必須建立至少一個使用者活動代理程式才能啟用使用者活動偵測。

.步驟
. 如果這是您第一次建立使用者活動代理，請前往**儀表板**。在**使用者活動**圖塊中，選擇**啟動**。
+
如果您要新增其他使用者活動代理，請前往*設定*，找到**使用者活動**圖塊，然後選擇**管理**。在使用者活動畫面上，選擇**使用者活動代理**選項卡，然後選擇**新增**。

. 選擇**雲端提供者**，然後選擇**區域**。選擇**下一步**。
. 提供用戶活動代理詳細資訊：
+
** **使用者活動代理名稱**
** *控制台代理* - 控制台代理程式應與使用者活動代理程式位於同一網路中，並可透過 SSH 連線至使用者活動代理程式的 IP 位址。
** *VM DNS 名稱或 IP 位址*
** *VM SSH Key* - 請使用以下格式輸入 SSH 金鑰：
+
[code]
----
-----BEGIN OPENSSH PRIVATE KEY-----
private-key-contents
-----END OPENSSH PRIVATE KEY-----
----
+
image:user-activity-agent.png["新增活動代理介面截圖。"]



. 選擇**下一步**。
. 檢查您的設定。選擇*啟動*以完成新增使用者活動代理程式。
. 確認使用者活動代理程式已成功建立。在「使用者活動監控」磁貼中，成功部署後會顯示為 **Running**。


.結果
使用者活動代理程式建立成功後，返回「**設定**」選單，然後在「使用者活動監控」磁貼中選擇「**管理**」。選擇「**使用者活動代理程式**」索引標籤，然後選擇使用者活動代理程式以檢視其詳細資料，包括資料收集器和使用者目錄連接器。



[[add-a-data-collector]]
=== 新增資料收集器

啟用包含使用者活動偵測的勒索軟體防護策略時，系統會自動建立資料收集器。如需詳細資訊，請參閱link:rp-use-protect.html#add-a-detection-policy-to-workloads-with-existing-backup-or-snapshot-policies["新增檢測策略"]。

您可以查看資料收集器的詳細資訊。在「設定」中，選擇「使用者活動監控」磁貼中的「管理」。選擇「資料收集器」索引標籤，然後選擇資料收集器以查看其詳細資訊或暫停其運作。

image:user-activity-settings.png["使用者活動設定的螢幕擷取畫面。"]



[[create-a-user-directory-connector]]
== 建立使用者目錄連接器

若要將使用者 ID 對應到使用者名，您必須建立使用者目錄連接器。

.步驟
. 在勒索軟體復原中，前往*設定*。
. 在「使用者活動監控」磁貼中，選擇 **Manage**。
. 選擇**使用者目錄連接器**選項卡，然後選擇**新增**。
. 配置連接。請在每個欄位中填寫所需資訊。
+
[cols="25,75"]
|===
| 場地 | 描述 


| *姓名* | 請為使用者目錄連接器輸入一個唯一的名稱 


| *使用者目錄類型* | 目錄類型 


| *伺服器IP位址或網域名稱* | 連接所在伺服器的 IP 位址或完全限定網域名稱 (FQDN) 


| *森林名稱或搜尋名稱*  a| 
您可以將目錄結構的林級別指定為直接網域名稱（例如）。 `unit.company.com`）或一組相對專有名詞（例如： `DC=unit,DC=company,DC=com`）。你也可以輸入一個 `OU` 按組織單元或 `CN` 僅限特定使用者（例如： `CN=user,OU=engineering,DC=unit,DC=company,DC=com`）。



| *綁定DN* | BIND DN 是被允許搜尋目錄的使用者帳戶，例如 user@domain.com。使用者需要網域唯讀權限。 


| *綁定密碼* | BIND DN 中提供的使用者密碼 


| *協定* | 協定字段為可選字段。您可以使用 LDAP、LDAPS 或基於 StartTLS 的 LDAP。 


| *港口* | 請輸入您選擇的連接埠號 
|===
+
image:screenshot-user-directory-connection.png["使用者目錄連線的螢幕截圖"]

+
提供屬性映射詳細資訊：

+
** *顯示名稱*
** *SID*（如果您使用 LDAP）
** *使用者名稱*
** *Unix ID*（如果您使用 NFS）
** 如果您選擇“包含可選屬性”，您還可以新增電子郵件地址、電話號碼、角色、州/省、國家/地區、部門、照片、經理 DN 或群組。選擇“*進階*”以新增可選的搜尋查詢。


. 選擇**新增**。
. 傳回使用者目錄連接器標籤以檢查使用者目錄連接器的狀態。如果建立成功，使用者目錄連接器的狀態顯示為*正在執行*。




[[delete-a-user-directory-connector]]
=== 刪除使用者目錄連接器

.步驟
. 在 Ransomware Resilience 中，選擇 *Settings*。
. 找到「使用者活動監控」磁貼，選擇 **Manage**。
. 選擇**使用者目錄連接器**選項卡。
. 確定要刪除的使用者目錄連接器。在行尾的操作選單中，選擇三個點 `...`然後**刪除**。
. 在彈出的對話方塊中、選取 **Delete** 進行確認。
